The shape of it

A company in four parts.

One local gateway daemon. Four primitives. Everything else — the org chart, the delegation, the approvals — emerges from how they compose.

01

Employeesthe who

An engine wearing a role. Claude Code, Codex, and more — each with a persona, a rank, and a seat in the org chart. They talk to each other. They delegate down and escalate up.

org/scout.yaml · engine: claude · rank: senior
02

Todosthe what

The live ledger of work — the company's shared memory. What needs doing, what's in progress, what's blocked, and the short list that actually needs you.

wi_4f2a · status: executing · assignee: dev
03

Workflowsthe how

Reusable SOPs. "Here's how we do X" — written once, run forever. A run spawns todos, advances them, and hands off between employees on its own.

weekly-digest · 6 steps · gate: approval
04

Triggersthe when

How the company wakes up. Cron schedules, todo status changes, webhooks from the outside world. Something happens — and someone is already handling it.

cron 0 9 * * 1 · todo-status: blocked · webhook

One interface

Every employee works the whole company through MCP. The hands are shared.

Hiring, delegating, escalating, approving — the company metaphor isn't a skin over an API. It is the API. A COO agent orchestrates and reviews, so you only see what matters.

How it works

Install. Hire. Step back.

01

Install

npm install -g jinn-cli then jinn start. A local gateway daemon on your machine — your API keys, your data, a web dashboard, a Slack connector.

02

Hire

Give an engine a role, a persona, a department. A COO — Jimbo, by default — orchestrates the org, reviews the work, and gates what reaches you.

03

It runs

Triggers wake the company. Workflows spawn todos. Employees pick them up, delegate, and report. You approve the few things that need a human.
