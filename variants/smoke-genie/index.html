The summoning

Every jinn answers to a rite.
Yours answers to four words.

One local daemon. Four primitives. That's the whole company — everything else is employees doing their jobs.

I · The vessels

Employees

An engine wearing a role. Claude Code, Codex, and more — each given a name, a rank, and a seat in the org chart. They talk to each other. They delegate. They escalate when it matters.

II · The ledger

Todos

The live work ledger — what needs doing, what's in flight, what's blocked, what needs you. The company's shared memory, visible at a glance.

III · The rites

Workflows

"Here's how we do X" — written down once, run forever. Reusable SOPs that spawn and advance Todos every time they run.

IV · The awakening

Triggers

How the company wakes: cron schedules, todo status changes, webhooks from the outside world. It works while you don't.

IIIIIIIV
One interface

MCP is the hands.

Every employee operates the whole company through one tool interface — hire, delegate, escalate, approve. A COO agent orchestrates and gates approvals, so you only see what matters.

The rite of passage

Install. Summon. Let it run.

01

Install

One package. No cloud account, no signup — the daemon lives on your machine.

npm install -g jinn-cli
02

Summon

Start the gateway. You get a web dashboard — chat, org chart, todos, workflows — and a Slack connector.

jinn start # the lamp is lit
03

Let it run

Hire employees, define workflows, set triggers. The company works around the clock; you approve what matters.

# hire, delegate, approve
