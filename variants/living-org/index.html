How we do things

Workflows

When it wakes

Triggers

How the company wakes up. Cron schedules, todo status changes, webhooks from the outside world. Work starts while you sleep; the COO reviews it before it reaches you.

One interface

MCP is the hands.

Every employee operates the whole company through one tool interface — reading the org, moving Todos, running Workflows, waking each other. A COO agent orchestrates, reviews, and gates approvals, so the human operator only sees what matters.

Proceedings

Install. Hire.
It runs.

i.

Install the gateway

npm install -g jinn-cli && jinn start — one local-first daemon. Your machine, your API keys, your data.

ii.

Hire your employees

Pick engines, write personas, set ranks and departments. The company metaphor is the API: hire, delegate, escalate, approve.

iii.

Watch it work

A web dashboard for chat, org chart, todos, workflows and sessions — plus a Slack connector. Triggers keep it running around the clock.
