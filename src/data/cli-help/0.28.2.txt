$ jinn --help
Usage: jinn [options] [command]

Lightweight AI gateway daemon

Options:
  -V, --version                output the version number
  -i, --instance <name>        Target a specific instance (default: jinn)
  -h, --help                   display help for command

Commands:
  setup [options]              Initialize Jinn and install dependencies
  start [options]              Start the gateway daemon
  stop [options]               Stop the gateway daemon
  restart [options]            Restart the gateway (detached — safe to run from
                               inside a session)
  status                       Show gateway status
  pair [options]               Create a one-time code for pairing another
                               browser
  unpair [options] [deviceId]  List paired browsers or unpair one by id
  limits [options]             Show engine rate limits, quota windows, and model
                               capabilities
  create [options] <name>      Create a new Jinn instance
  list                         List all Jinn instances
  remove [options] <name>      Remove a Jinn instance from the registry
  nuke [name]                  Permanently delete a Jinn instance and all its
                               data
  migrate [options]            Print the canonical pending instance-migration
                               prompt or complete a verified migration
  workflow                     Operate workflow definitions and runs
  skills                       Manage skills from the skills.sh registry
  help [command]               display help for command

Options:
  --force     Delete existing home dir and reinitialize from scratch

$ jinn start --help
Usage: jinn start [options]

Start the gateway daemon

Options:
  --daemon           Run in background
  -p, --port <port>  Override the gateway port from config
  --take-port        Take over a port owned by another Jinn instance
  -h, --help         display help for command

$ jinn stop --help
Usage: jinn stop [options]

Stop the gateway daemon

Options:
  -p, --port <port>  Port to kill the process on (default: from config or 7777)
  --take-port        Stop a process on the target port even when it belongs to
                     another Jinn instance
  -h, --help         display help for command
